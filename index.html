This website provides information about the UE5 Stone Texture Generator, a Python application built with CustomTkinter and NumPy for creating realistic, seamless stone textures using procedural generation techniques. The textures are designed for use in game engines like Unreal Engine (UE5) or other 3D applications.

üíª How It Works

The core of the application is the StoneGeneratorEngine class, which leverages vectorized NumPy operations for high-performance texture generation. It produces the four essential maps needed for Physically Based Rendering (PBR): Albedo, Normal, Roughness, and Height.

1. The Generation Process

The textures are based on fractal noise, specifically a technique related to Perlin noise or Value noise combined with Fractal Brownian Motion (FBM).

  • Noise Layer Generation: The generate_noise_layer method creates a base layer by summing multiple layers (called octaves) of noise. Each octave has a higher frequency and lower amplitude than the last (controlled by lacunarity and persistence), adding fine detail to the overall shape.
  • Base Height Map: This serves as the fundamental structure of the stone.
  • Domain Warping: To create more organic and less grid-like patterns, the coordinates used to sample the noise are warped or offset by a secondary, lower-frequency noise function. This introduces natural-looking bends, folds, and river-like features often seen in eroded terrain or rock.

# structure = (base_noise + 0.08 warp_x + 0.08 warp_y)

Contrast Refinement: The resulting structure is passed through a simple contrast adjustment to sharpen the rock features.

2. Output Maps (PBR Textures)

The application generates the following four essential maps for rendering:

Map Purpose Generation Method
Albedo (Color) Defines the base color and diffuse light reflection. Blends two user-defined colors (color1 and color2) based on the value of the Height Map (structure).
Normal Simulates small-scale surface detail and bumps using fake lighting. Calculated using the gradient of the Height Map (structure). The x and y derivatives are scaled by the depth_strength parameter and then used to construct the x, y, z vectors of the normal map, which are then normalized.
Roughness Controls how sharp or blurry reflections are (microsurface detail). Derived directly from the Height Map, where higher areas (peaks) are generally made rougher (lighter values).
Height (Displacement/Bump) A grayscale image representing surface height (black is low, white is high). The normalized structure array is scaled from 0 to 255.

The application features several sliders and inputs to control the resulting texture:

  • Resolution: Output size (e.g., 2048x2048, 4096x4096). The preview uses a fixed 1024x1024 resolution for speed.
  • Noise Scale (Zoom): Controls the overall size of the rock features. A smaller scale means larger features.
  • Detail Intensity (Contrast): Adjusts the value range of the base structure, sharpening the difference between the peaks and valleys.
  • 3D Depth Strength: Multiplies the gradient vectors before normalizing the Normal Map, controlling the perceived height/depth of the bumps.
  • Micro-Details (Octaves): The number of noise layers (octaves) summed to create the fractal noise. More octaves add finer detail but increase generation time.
  • Seed: An integer used to initialize the random number generator, ensuring the texture can be perfectly reproduced.
